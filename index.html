Introduction

Hello NITR is a Flutter-based mobile application designed specifically for the National Institute of Technology Rourkela (NITR) community. The app aims to streamline communication and contact sharing among students, faculty, and staff, providing a centralized platform for official use. With features like user authentication, contact management, and profile viewing, Hello NITR serves as a comprehensive tool to enhance connectivity within the institute.

Project Overview

  • Name: hello_nitr
  • Description: A contact-sharing app for NIT Rourkela official use.
  • Version: 2.0.1
  • Environment: Dart SDK >=3.4.0 <4.0.0

Project Structure

The application follows a modular Model-View-Controller (MVC) architecture, enhanced with state management using the Provider package and utility components for scalability and maintainability.

Key Components

Let’s dive into the key components that make up the application:

Models

  • **LoginResponse**: Represents the data received after a successful login attempt, including status codes and user information.
  • **User**: Defines the user model, encapsulating properties like name, email, department, and contact details.

Controllers

  • **LoginController**: Manages user authentication and session management.
  • **PinCreationController**: Handles the creation and validation of user PINs for added security.
  • **PinUnlockScreenController**: Manages PIN verification and supports biometric authentication for app unlocking.
  • **OtpVerificationController**: Handles OTP generation, sending, and verification processes.
  • **ContactsUpdateController**: Manages the fetching and updating of contact information from the server.
  • **UserProfileController**: Oversees user profile data retrieval and updates.
  • **SimSelectionController**: Handles SIM card selection and validation, especially for OTP verification.
  • **SplashScreenController**: Manages the initial loading procedures and login status checks when the app starts.

Running the Application

Prerequisites

Before you begin, ensure you have met the following requirements:

  • Flutter SDK: Version >=3.4.0 <4.0.0
  • Dart SDK: Compatible with your Flutter SDK version
  • Development IDE: Android Studio, Visual Studio Code, or any IDE with Flutter and Dart plugins
  • Platform SDKs:
    • Android SDK: For Android development
    • Xcode: For iOS development (required for building on macOS)
  • Connected Device or Emulator: An Android device/emulator or iOS device/simulator

Setup Instructions

  1. Clone the Repository

    git clone https://github.com/yourusername/hello_nitr.git
cd hello_nitr

  2. Install Dependencies

    flutter pub get

  3. Configure the Environment

    • Ensure your Flutter SDK is properly set up.

    • Verify connected devices:

      flutter devices

  4. Run the Application

    • On Android

      flutter run

    • On iOS

      flutter run

    • Specify a Device

      flutter run -d <device_id>

Building the Application

Building for Android

  1. Generate a Keystore

    keytool -genkey -v -keystore ~/hello_nitr_keystore.jks -storetype JKS -keyalg RSA -keysize 2048 -validity 10000 -alias hello_nitr

  2. Create **key.properties**

    In the project root, create a file named key.properties:

    storePassword=your_store_password
keyPassword=your_key_password
keyAlias=hello_nitr
storeFile=/path/to/hello_nitr_keystore.jks

  3. Update **build.gradle**

    Configure signing configs in android/app/build.gradle:

    def keystoreProperties = new Properties()
def keystorePropertiesFile = rootProject.file('key.properties')
if (keystorePropertiesFile.exists()) {
    keystoreProperties.load(new FileInputStream(keystorePropertiesFile))
}

android {
    // ...

    signingConfigs {
        release {
            keyAlias keystoreProperties['keyAlias']
            keyPassword keystoreProperties['keyPassword']
            storeFile file(keystoreProperties['storeFile'])
            storePassword keystoreProperties['storePassword']
        }
    }

    buildTypes {
        release {
            signingConfig signingConfigs.release
        }
    }
}

  4. Build the APK

    flutter build apk --release

  5. Locate the APK

    Find your APK at:

    build/app/outputs/flutter-apk/app-release.apk

Building for iOS

Important Note: SIM card selection is not supported on iPads and some iOS devices. On iOS, the app defaults to manual phone number input for OTP verification. Please refer to the Known Issues and Limitations section for more details.

  1. Switch to the **ios-release** Branch

    The ios-release branch contains the necessary adjustments for iOS devices:

    git checkout ios-release

  2. Open the Project in Xcode

    open ios/Runner.xcworkspace

  3. Configure Code Signing

    • Select your development team in the Signing & Capabilities tab.
    • Ensure that the bundle identifier is unique.

  4. Update **Info.plist**

    • Ensure all necessary permissions are included.
    • Add any required privacy descriptions.

  5. Build the App

    From the command line:

    flutter build ios --release

  6. Archive and Distribute

    • In Xcode, go to Product > Archive.
    • Follow the on-screen instructions to distribute your app.

Key Features and Workflows

Authentication

Login Process

  • User Input

    Users enter their credentials on the login screen:

    String username = _usernameController.text;
String password = _passwordController.text;

  • Authentication

    The LoginController handles authentication:

    LoginController loginController = LoginController();
loginController.login(username, password);

PIN Creation and Verification

  • PIN Creation

    After successful login:

    PinCreationController pinController = PinCreationController();
pinController.createPin(pin);

  • PIN Verification

    On app launch:

    PinUnlockScreenController pinUnlockController = PinUnlockScreenController();
pinUnlockController.verifyPin(enteredPin);

  • Biometric Authentication

    Optional biometric authentication using local_auth:

    LocalAuthentication auth = LocalAuthentication();
bool authenticated = await auth.authenticate(
  localizedReason: 'Please authenticate to access the app',
  options: const AuthenticationOptions(biometricOnly: true),
);

OTP Verification

  • OTP Generation and Verification

    OtpVerificationController otpController = OtpVerificationController();
otpController.sendOtp(phoneNumber);
// After user enters OTP
otpController.verifyOtp(enteredOtp);

Known Issues and Limitations

SIM Card Selection on iPads and iOS Devices

  • Issue: SIM card selection is not supported on iPads and some iOS devices due to platform limitations.
  • Impact: Users cannot automatically retrieve their phone number for OTP verification.
  • Workaround: On iOS, the app defaults to manual phone number input for OTP verification.

Important: Ensure that users on iOS devices are informed to manually enter their phone numbers during the OTP verification process.

Branches and Source Code
